# 核心玩法与机制设计

> 本文档聚焦游戏的 **核心玩法循环** 和 **差异化机制**，定义"异次元矩阵"这款游戏"好玩"的根基。

---

## 一、设计理念

迷宫不是静态的地图，而是一个 **正在运行的巨型魔方机械**。玩家不是在"走迷宫"，而是在"与迷宫博弈"。三大核心支柱：

| 支柱 | 关键词 | 提供的体验 |
|------|--------|-----------|
| 空间博弈 | 房间置换、维度坍缩 | 随机性的"广度"——每局路径不同 |
| 环境变量 | 物理常数异化 | 策略性的"深度"——因地制宜 |
| 身体代价 | 模块化义体改造 | 养成性的"情感粘性"——我的角色独一无二 |

> **核心公式：空间随机性 × 环境变量 × 身体改造 = 无穷的局内组合 = 极高的重玩价值**

---

## 二、核心机制详解

### 2.1 维度坍缩与房间置换（Room Shifting）

#### 基础概念

迷宫由 N×N×N 的房间组成，房间每隔一段时间（或由玩家触发）进行 **物理位置交换**。

#### 核心玩法点

- **预判与观察**：玩家可通过房间墙壁上的"编号"、"齿轮声"或光线闪烁频率预判下一次置换。熟悉规律的老手可以准确预测 2-3 步之后的地图状态。
- **动态寻路**：路径不是死板的，玩家需要利用置换逻辑，像拼图一样把通往出口的房间"拼"到自己面前。这让传统的"寻路"变成了"空间规划"。
- **战术驱逐**：无法击败强敌？将其引诱到即将"离位"的房间，通过机关将该房间送往迷宫边缘甚至"粉碎区"。
- **主动操控（高级技能）**：特定英雄技能或道具可以 **主动操控** 邻近房间的置换，将敌人队伍传送到高难度怪物房，或将自己置换到资源丰富的中心区。

#### 动态缩圈机制

传统大逃杀的"毒气缩圈"在矩阵中表现为 **空间的物理抹除**：

- 随着时间推移，外围房间发生"维度坍缩"直接消失
- 被坍缩吞没的玩家直接淘汰
- 剩余空间越来越小，对抗越来越激烈
- 玩家可以利用置换技能将房间从坍缩区"抢救"出来，或反过来加速某个方向的坍缩

#### 跨房间连锁反应

利用服务端权威的优势，设计跨房间的因果链：
- 在 A 房间拉动电闸 → C 房间水位上涨
- 在 B 房间激活磁场 → D 房间所有金属模块失灵
- 所有玩家看到的状态绝对同步，杜绝作弊

### 2.2 物理常数异化（Anomalous Physics）

#### 基础概念

每个房间不仅位置不同，其 **物理法则** 也可以截然不同。每个房间随机生成一个或多个物理词条。

#### 词条示例

| 词条名称 | 效果 | 战术影响 |
|---------|------|---------|
| **失重状态** | 战斗变为 360° 全向，投掷物不再下坠 | 近战英雄劣势，射击英雄优势 |
| **时间滞后** | 所有动作延迟 2 秒发生 | 必须进行预判操作，策略大于反应 |
| **光影杀机** | 阴影中安全，接触光线持续掉血（或反之） | 迫使玩家控制光源、利用掩体 |
| **镜像投射** | 出现动作完全相反的镜像分身 | 需要操控角色避开镜像触碰陷阱 |
| **极低气压** | 声音传播受限，喷射类能力增幅 | 隐蔽性提高，喷射模块成为核心 |
| **无阻力** | 投掷物无限飞行直到命中 | 可实现"跨屏击杀"，远程武器极强 |
| **禁魔区** | 能量类技能无法使用 | 纯物理近战/投掷流称王 |
| **时间加速** | 所有效果持续时间减半，冷却减半 | 高频操作、快节奏战斗 |
| **磁场失控** | 金属模块受损/受干扰 | 生化模块流派的主场 |

#### 组合化学反应

物理词条与身体模块之间可以产生 **隐藏的超载组合（Overload）**：
- 极低气压 + 高压喷射腿 → 位移距离翻倍 + 音爆伤害
- 失重 + 磁力抓钩 → 可在房间中自由飞行
- 时间加速 + 过载核心 → 短时间内爆发力极强但有自毁风险

> **设计目标**：让玩家沉迷于发现这些隐藏组合，就像《以撒的结合》中发现道具协同一样上瘾。

### 2.3 感官借用与生态追踪（Sensory Linking）

#### 基础概念

在密闭的矩阵中，**信息比武器更重要**。玩家拥有"频率调整器"，可短时间同步其他生物或陷阱的感官。

#### 玩法点

| 能力 | 效果 | 战术价值 |
|------|------|---------|
| **借眼侦查** | 通过隔壁巡逻机器人的视角观察陷阱布局 | 零风险探路 |
| **声纳定位** | 失去视觉，但声波显示墙壁后的移动物体 | 反隐身、预判敌人位置 |
| **痛觉同步** | 将受伤反馈同步给敌人 | 换血流战术——以伤换伤 |
| **记忆残留** | 读取房间中残留的前任探索者记忆 | 获取陷阱线索、隐藏通道信息 |

#### 信息战博弈

- 高端玩家会故意释放 **虚假感官信号** 来误导对手
- 某些房间词条可能让感官链接变得不稳定或失真
- "信息获取"本身有风险——链接期间自身处于脆弱状态

---

### 2.4 模块化义体改造（Modular Augmentation）

#### 基础概念

废弃传统的商店/装备系统。击败敌人后，玩家必须 **拆卸自己的零件** 来安装敌人的"模块"。每一次改造都有 **代价**。

#### 核心设计：代价转换

| 你想安装的 | 你必须拆掉的 | 游戏影响 |
|-----------|-------------|---------|
| 热成像眼 | 语言模块 | 无法阅读任何文字提示 |
| 高速机械腿 | 装甲模块 | 移动极快但一碰就碎 |
| 全身刀刃覆甲 | 灵活关节 | 强力近战但移速极慢、无法拾取远程武器 |
| 人工肺 | 攻击模块之一 | 能穿越毒气房但输出降低 |
| 引力反转模块 | 方向感知 | 可飞行但操控方向混乱 |

#### 极端流派与局内转职

- 每局开始只有基础技能，"杀敌取件"获得模块
- 可以把法师英雄强行改造成近战肉盾，只要抢到足够装甲模块
- 这种 **"局内转职"** 让每一局体验完全不同
- 参考 Roguelike 的构筑乐趣，但发生在多人竞技场景中

#### 环境适应性改造

某些房间的物理词条会迫使玩家临时改造：
- 毒气房 → 需要"人工肺"模块
- 高温房 → 需要散热装置
- 磁场房 → 金属模块失灵，需转向生化模块

> **设计哲学**：让玩家在"变强"和"付出代价"之间做选择，每次改造都是一个 **有意义的决策**。

## 三、核心游戏循环

```
进入矩阵
  │
  ▼
探索房间 ←──────────────────────────┐
  │                                   │
  ▼                                   │
感知环境（物理词条？陷阱？敌人？）      │
  │                                   │
  ├── 安全 → 搜刮模块/资源 ──────────┤
  │                                   │
  ├── 陷阱 → 利用感官借用侦查         │
  │          → 选择绕行/破解/引诱敌入  │
  │                                   │
  ├── 遭遇敌人 → 战斗                 │
  │   ├── 胜利 → 拆卸模块 → 改造抉择  │
  │   └── 不敌 → 利用房间置换逃脱 ────┤
  │                                   │
  └── 房间置换事件触发                 │
      → 重新评估位置与路线 ───────────┘
  │
  ▼
维度坍缩压缩空间 → 最终对决 → 胜者逃离矩阵
```

## 四、与传统 MOBA / 大逃杀的差异化对比

| 维度 | 传统 MOBA | 传统大逃杀 | 异次元矩阵 |
|------|----------|-----------|-----------|
| 地图 | 固定三路 | 固定大地图+缩圈 | **动态置换迷宫 + 空间坍缩** |
| 成长 | 商店买装备 | 捡装备 | **杀敌取件 + 代价改造** |
| 环境 | 被动背景 | 地形掩体 | **物理词条主动影响战斗** |
| 信息 | 小地图视野 | 听声辨位 | **感官借用 + 信息战博弈** |
| 空间 | 二维平面 | 三维地形 | **三维迷宫 + 房间可移动** |
| 缩圈 | 无 | 毒圈/轰炸区 | **维度坍缩（物理消失）** |

## 五、MVP 验证清单

以下是核心玩法 MVP 需要验证的关键假设：

| # | 假设 | 验证方式 | 优先级 |
|---|------|---------|--------|
| 1 | 房间置换机制直观有趣 | 用白模做 3×3 迷宫，手动触发置换，观察试玩反应 | ⭐⭐⭐ |
| 2 | 模块改造的"代价感"有决策张力 | 给 3-4 种模块 + 2 种代价，看玩家是否纠结 | ⭐⭐⭐ |
| 3 | 物理词条让战斗有变化 | 实现 2 种词条（失重 + 无阻力），对比无词条体验 | ⭐⭐ |
| 4 | 维度坍缩产生紧迫感 | 实现基础缩圈，观察节奏是否合适 | ⭐⭐ |
| 5 | 多人联网下玩法依然成立 | 2-4 人联网测试 | ⭐⭐⭐ |
